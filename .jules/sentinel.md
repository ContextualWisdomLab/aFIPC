## 2024-06-27 - [Unbounded Recursion on User Input]
**Vulnerability:** The functions checking user inputs (`checkCorrect`, `checkoldformBILOGprior`, `checknewformBILOGprior`) used recursion to repeatedly prompt for input if the input didn't match validation regex. In non-interactive contexts (like automated CI/CD pipelines without proper standard input handling or when users continuously provide bad input), this recursive calling would cause a "C stack usage is too close to the limit" error and crash the R session, leading to a Denial of Service (DoS) vulnerability.
**Learning:** Functions designed for user interaction should never rely on unbounded recursion for retry logic. The R interpreter has a limited call stack limit, and recursive prompts without an upper bound are a systemic vulnerability in R CLI scripts.
**Prevention:** Instead of using recursion for retries, always use a loop (`while` or `for`) and enforce a hard limit on the number of attempts (e.g., `attempts < 5`). If the maximum attempt limit is reached, gracefully stop execution with a clear error message.
