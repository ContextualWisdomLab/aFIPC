## 2024-06-25 - [Fix infinite recursion DoS in non-interactive sessions]
**Vulnerability:** Found a Denial of Service (DoS) vulnerability in `aFIPC.R` where recursive wrappers around `readline()` (e.g., `checkCorrect`, `checkoldformBILOGprior`, `checknewformBILOGprior`) caused an infinite loop and eventually a C stack overflow when the package is run in a non-interactive environment (like CI/CD pipelines, background jobs, or cron). `readline()` returns an empty string `""` on EOF in headless environments, constantly failing the `grepl("^[0-9]+$", n)` regex check and recursing endlessly.
**Learning:** Functions that rely on interactive user input via stdin must account for non-interactive execution contexts. Failing to do so can lead to resource exhaustion (DoS) or unexpected crashes when deployed to automated systems.
**Prevention:** Always check if the current session is interactive (e.g., `interactive()` in R) before prompting for input, and provide sensible defaults or error out gracefully if non-interactive. Avoid recursive retry logic for user inputs without setting a maximum recursion depth.
