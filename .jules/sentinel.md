## 2024-06-26 - [CRITICAL] Prevent Infinite Recursion DoS in Interactive Prompts
**Vulnerability:** Found a vulnerability in `R/aFIPC.R` where nested functions using `readline()` (`checkCorrect`, `checkoldformBILOGprior`, and `checknewformBILOGprior`) recursively called themselves upon receiving invalid input or EOF. In non-interactive/batch environments (like CI pipelines or background jobs), `readline()` returns an empty string `""`, leading to an infinite recursion loop and causing a Denial of Service (C stack usage limit crash).
**Learning:** Legacy CLI prompt patterns often fail dangerously in non-interactive batch modes. It's critical to ensure functions that expect interactive input explicitly check for `interactive()` and fail safely if they're executing in a non-interactive context.
**Prevention:** Always validate if a session is interactive using `interactive()` before invoking `readline()`, and use `stop()` or supply a safe default rather than allowing infinite recursive reprompting in automated environments.
